# MiniFlash: Minimal Block-Sparse Row (BSR) Attention Specification

**Objective:** Implement a hardware-aligned, load-balanced Block-Sparse Attention module using PyTorch (for planning/routing) and OpenAI's Triton (for GPU execution). 

**Context:** The model (Chronos-2) requires Group Attention across the batch axis. Standard dense matrices ($B \times B$) cause OOM errors. This implementation uses a $16 \times 16$ Block-Sparse Row (BSR) approach, flattening the time and batch dimensions to maximize GPU occupancy.

---

## 1. Architectural Constants & Target Shapes
* **Hardware Block Size (`BLOCK_SIZE`):** `16` (Matches NVIDIA Tensor Core `mma.sync` minimums).
* **Input Shape:** `(Batch, Time, Dim)` $\rightarrow$ transposed to `(Time, Batch, Dim)`.
* **Target Flattened Shape:** `(Time * Padded_Batch, Heads, Head_Dim)`.
* **Workspace Size:** `(Num_Tiles, Heads, Head_Dim)` for intermediate float32 accumulations.

---

## Phase 1: BSR Data Formatter (PyTorch)
**Goal:** Prepare and flatten the tensors so the GPU sees one continuous sequence.

**Requirements:**
1.  **Pad Batch:** Calculate `pad_len = (16 - (B % 16)) % 16`. Concatenate zero-tensors to $Q, K, V$ along the batch dimension so `B_padded` is a multiple of 16.
2.  **Flatten:** Reshape $Q, K, V$ from `(T, B_padded, H, D)` to `(T * B_padded, H, D)`.
3.  **Generate Diagonal Metadata:** * Calculate `total_blocks = T * (B_padded // 16)`.
    * Generate `indptr` as a 1D tensor from `0` to `total_blocks + 1`.
    * Generate `indices` as a 1D tensor from `0` to `total_blocks`.

---

## Phase 2: Load Balancing Planner (PyTorch)
**Goal:** Prevent GPU stragglers by mapping varying row workloads into equal-sized execution tiles.

**Requirements:**
1.  Define `BLOCKS_PER_TILE = 16`.
2.  Calculate total required compute blocks from `indptr`.
3.  Calculate `num_tiles = cdiv(total_compute_blocks, BLOCKS_PER_TILE)`.
4.  Generate `task_to_row` (Shape: `[total_compute_blocks]`):
    * Create `flat_task_ids = torch.arange(total_compute_blocks)`.
    * Use `torch.searchsorted(indptr, flat_task_ids, right=True) - 1` to map every block to its corresponding row.

---

## Phase 3: Executor - Pass 1 (Triton)
**Goal:** Compute attention partials and Online Softmax statistics.

**Kernel Signature:** `@triton.jit def bsr_fwd_pass1_kernel(Q, K, V, indptr, indices, task_to_row, workspace_v, workspace_m, workspace_l, ...)`

**Execution Logic:**
1.  **Grid:** `(num_tiles, Heads)`.
2.  **Task Range:** `tile_id = tl.program_id(0)`. Calculate `task_start` to `task_end` based on `BLOCKS_PER_TILE`.
3.  **Initialize Partials:** `m_i = -inf`, `l_i = 0`, `acc = 0`.
4.  **Routing Loop:** Loop through assigned tasks.
    * Look up `row_idx = tl.load(task_to_row + task)`.
    * Look up `col_idx = tl.load(indices + task)`.
    * Load $16 \times D$ blocks of $Q$, $K$, $V$.
    * Compute $S = Q \times K^T$.
    * Update `m_i`, `l_i`, and `acc` (Online Softmax).
5.  **Workspace Write:** Write `acc`, `m_i`, and `l_i` to the global workspace buffers at the index corresponding to the specific `tile_id` and `row_idx`.

---

## Phase 4: Executor - Pass 2 (Triton)
**Goal:** Reduce partial computations across split rows into the final output.

**Kernel Signature:**
`@triton.jit def bsr_fwd_pass2_kernel(Out, workspace_v, workspace_m, workspace_l, locks, ...)`

**Execution Logic:**
1.  **Grid:** `(Total_Flattened_Rows // 16, Heads)`.
2.  **Combine:** For a given row, read all partial `m_i` and `l_i` values from the workspace generated by different tiles.
3.  **Final Softmax:** Calculate the global maximum $M$ and global sum $L$.
4.  **Scale & Accumulate:** Rescale the partial `acc` vectors using $\exp(\text{partial\_m} - M)$ and sum them. Divide by the global $L$.
5.  **Write:** Store the final normalized $16 \times D$ block to the `Out` tensor.

---

## Phase 5: `nn.Module` Wrapper
**Goal:** Provide a drop-in replacement for standard Multi-Head Attention.

**Requirements:**
1.  **Init:** Initialize $Q, K, V, O$ linear layers and allocate an empty persistent `workspace` tensor (e.g., 32MB).
2.  **Forward:**
    * Unpack `bsr_metadata`.
    * Apply Phase 1 (Pad & Flatten).
    * Apply Phase 2 (Plan).
    * Launch Phase 3 (Pass 1 Triton Kernel).
    * Launch Phase 4 (Pass 2 Triton Kernel).
    * Slice off the padding, reshape back to `(Time, Batch, Head * Dim)`, apply the Output linear projection, and return.
